### 📱 MOBILE KUBERNETES INFRA STRUCTURE

> **'스팀덱 클라우드 인프라"**
> 스팀덱의 기본 운영체제 ARC LINUX상에 개발환경, 저장소, 빌드, 배포, 서비스를 구축했습니다.
> 노트북에도 가능하지만,스팀덱의 운영제체는 리눅스가 기본이여서 리눅스 가상화 없이 구축이 가능합니다.

---

#### 🏗️ 엔터프라이즈 기술 매핑 (Enterprise Tech Mapping)
단순한 설치를 넘어, 실제 기업에서 사용하는 솔루션들을 모바일 환경에 최적화하여 구성했습니다.

| 분류 | 적용 기술 | 기업용 솔루션 대응 및 역할 |
| :--- | :--- | :--- |
| **REPO** | **Gitea** | **GitLab / GitHub (내부 코드 저장소)** <br> 외부 의존성 없이 기기 내에서 소스 코드를 직접 관리 및 보관 |
| **CI/CD** | **ArgoCD** | **AWS Pod Auto-Scaling / CI·CD** <br> 코드 변경 시 시스템이 스스로 감지하여 서비스를 자동 배포/확장 |
| **FLATFORM** | **K3s** | **Kubernetes / Docker (표준 컨테이너 엔진)** <br> 다수의 서비스를 유기적으로 관리하는 기업 표준 경량화 엔진 |
| **MONITOR** | **Prometheus** | **Jennifer, Grafana, Datadog / 시스템 통합 대시보드** <br> 서버 부하, 온도, 서비스 가용성을 실시간으로 시각화 및 모니터링 |

---

#### 🚀 핵심 문제 해결 (Key Achievements)

**1. 하드웨어 제약 조건 극복 (Resource Optimization)**
* 일반적인 쿠버네티스 엔진은 메모리 점유율이 높아 모바일 기기에 부적합함.
* **K3s(경량 쿠버네티스)**를 채택하고 불필요한 기본 에이전트를 비활성화하여 시스템 가용성을 높임.

**2. SteamOS 환경 최적화**
* SteamOS의 읽기 전용(Read-only) 파일 시스템 특성상 표준 설치 불가.
* 시스템 잠금 해제 및 전용 볼륨 구성을 통해 재부팅 후에도 데이터가 유지되는 **영구 스토리지** 환경 구축.

**3. 완전 독립형 개발 환경 (Self-Hosted)**
* 외부 클라우드 도움 없이 기기 내부의 **Gitea(코드)**와 **ArgoCD(배포)**가 연동되어 기기 하나만으로 전체 서비스 생애 주기를 관리함.
